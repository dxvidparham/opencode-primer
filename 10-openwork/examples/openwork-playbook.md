# OpenWork Playbook

> 📌 **This is a hands-on companion to [Module 10: OpenWork Integration](../README.md).** Use `~/opencode-practice` for the guided path, then transfer to your own projects.

## Objectives

- Validate a local OpenWork workflow using your existing OpenCode habits
- Practice one collaboration-oriented scenario
- Create a repeatable adoption checklist for personal/team use

## Prerequisites

```bash
cd ~/opencode-practice
opencode
```

## Exercises

### Exercise 1: Migration Mindset Check

Prompt:

```
Given this repository, list which workflows map directly from OpenCode to OpenWork and which need adaptation
```

### Exercise 2: Collaboration Scenario

Prompt:

```
Simulate a teammate handoff: summarize current state, pending tasks, and recommended next actions in a shareable format
```

### Exercise 3: Integration Plan

Prompt:

```
Draft a 7-day adoption plan for OpenWork in a small team using this project as baseline
```

### Exercise 4 (Transfer): Personal Team Context

Repeat Exercise 3 with your real project constraints.

## Verification Checklist

- [ ] I can articulate OpenCode-to-OpenWork workflow mapping
- [ ] I can produce handoff-ready outputs for collaboration
- [ ] I have a practical rollout plan for my own context
