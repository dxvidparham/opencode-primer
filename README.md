## The Problem

You installed opencode. You ran a few commands. Now what?

- **The documentation describes features — but doesn't show you how to combine them.** You know opencode has tools, but not how to chain them with bash, file operations, and workflows that actually save hours.
- **There's no clear learning path.** Should you learn file operations before bash? Search tools before editing? You end up skimming everything and mastering nothing.
- **Examples are too basic.** A "hello world" example doesn't help you build a production code review pipeline that uses search, editing, and automation.

You're leaving 90% of opencode's power on the table — and you don't know what you don't know.

## Example Workflows

### Complete Code Review Workflow

```bash
# Uses: Search + File Operations + Bash

# 1. Search for potential issues
opencode search "TODO|FIXME|BUG" --include="*.{js,ts,py}"

# 2. Read files with issues
opencode read /path/to/file.js

# 3. Edit files to fix issues
opencode edit /path/to/file.js --old="TODO: fix this" --new="Fixed: implemented solution"

# 4. Run tests
opencode bash "npm test"
```

### Automated Refactoring

```bash
# Uses: Search + Edit + Task Automation

# 1. Find all occurrences of old function name
opencode search "oldFunctionName" --include="*.js"

# 2. Batch rename across files
for file in $(opencode search -l "oldFunctionName" --include="*.js"); do
  opencode edit "$file" --old="oldFunctionName" --new="newFunctionName" --replaceAll
done

# 3. Verify changes
opencode bash "npm run lint && npm run test"
```

### DevOps Deployment

```bash
# Uses: Automation + Bash + Workflows

# 1. Check environment
opencode bash "docker --version && node --version"

# 2. Build project
opencode bash "npm run build"

# 3. Deploy
opencode bash "npm run deploy"

# 4. Verify deployment
opencode bash "curl -s https://your-app.com/health"
```

## Best Practices

### Do's

- Start simple with basic commands
- Add features incrementally
- Use search before editing
- Test commands in safe environment first
- Document custom workflows
- Version control your automation scripts
- Share workflows with team

### Don'ts

- Don't run destructive commands without testing
- Don't hardcode sensitive information
- Don't skip error handling
- Don't over-complicate simple tasks
- Don't ignore security best practices
- Don't commit secrets or credentials

## Troubleshooting

### Command Not Working

1. Check command syntax and options
2. Verify file paths and permissions
3. Review opencode version
4. Test with simpler examples first

### File Operations Failed

1. Check file exists and is readable/writable
2. Verify exact string matching
3. Check for special characters
4. Test with backup first

### Search Not Finding Results

1. Check search pattern syntax
2. Verify file extensions in include pattern
3. Check search directory
4. Test with simpler pattern first
