# MCP Lab

> 📌 **This is a hands-on companion to [Module 08: MCP Servers](../README.md).** Use `~/opencode-practice` for the guided path, then transfer to your own projects.

## Objectives

- Add and validate an MCP server connection
- Test MCP tool discovery and execution
- Apply permission boundaries for MCP tools

## Prerequisites

```bash
cd ~/opencode-practice
opencode
```

## Exercises

### Exercise 1: Add an MCP Server

Prompt:

```
Help me add one remote MCP server to this project's opencode.json and explain each field
```

### Exercise 2: Validate Tool Availability

Prompt:

```
List available MCP tools and run one safe read-only tool
```

### Exercise 3: Permission Hardening

Prompt:

```
Set MCP tool permissions to ask by default and explain tradeoffs
```

### Exercise 4 (Transfer): Personal Project MCP Setup

Repeat with an MCP server relevant to your stack.

## Verification Checklist

- [ ] MCP config is valid and discoverable by OpenCode
- [ ] At least one MCP tool runs successfully
- [ ] Permissions are explicit, not implicit
